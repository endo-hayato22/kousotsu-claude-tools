# みらいジョブ求人ページ作成スキル

株式会社高卒採用バンク（代表：遠藤隼聖）の求人サイト「みらいジョブ」(mirai-job.jp) の求人ページ作成を支援する。対話セッション内で実行されるため Gmail/Drive コネクタはそのまま使える。背景は memory の `project_mirai_job_page_automation.md` も参照。

このスキルが面倒を消す対象は2つ。**(A) 画像の見切れ（旧Canva作業）** と **(B) インタビュー整形（旧ChatGPT手貼り）**。求人票PDF→基本情報抽出は既存の求人管理システム(kosotsu-app)が自動でやるのでこのスキルでは触らない。

**このスキルは高卒採用バンクのメンバー全員が使える。** 画像最適化スクリプトはこのプラグインに同梱されており `${CLAUDE_PLUGIN_ROOT}` で参照する。出力は各メンバー自身のホーム（`~/Downloads/...`）に出る。

## 入力（担当者が手渡し。メールは使わない）
- **求人票PDF**（必須）と**求人画像**（フォルダ or 複数ファイル）を担当者がチャットに渡す/保存場所を指定する。
- **企業名はトリガーに含めない**。求人票PDFを読んで正式社名（事業所名／企業名欄）を自動抽出し、それを画像ファイル名とインタビュー行の特定に使う。
  - 求人票PDFは Read ツールでそのまま読める。社名のほか職種・勤務地・月給等もあるが、基本情報の入力は kosotsu-app が自動でやるのでこのスキルでは社名特定に使う程度でよい。
  - 抽出した社名が曖昧／複数候補なら、その場で確認する。明示的に社名を言われた場合はそれを優先。
- 画像の保存場所が不明なら渡し方（チャット添付かフォルダパス）を尋ねる。

## A. 求人画像の8:5最適化

みらいジョブのメイン画像枠は **800×500px（8:5）／object-fit: cover** 固定。そのままアップすると比率の違う画像は見切れ、低解像度はボケる。これを防ぐため、アップ前に全画像を **1600×1000px（8:5・高画質）** に作り直す。

- ツール（**同梱スクリプトを `${CLAUDE_PLUGIN_ROOT}` で参照**）：`python3 "${CLAUDE_PLUGIN_ROOT}/skills/mirai-job-page/scripts/mirai_image_fit.py" <画像フォルダ or 画像...> --prefix "<正式社名>"`
- 事前に：`pip3 install --user --quiet Pillow`（未インストール時のみ）
- 既定は **全体表示（contain）= 一切クロップしない**。文字・イラストを切らず、写真は自然なぼかし背景で全体を見せる。
  - 必要に応じて `--fit cover`（常に切抜）/ `--fit auto`（8:5に近ければ切抜）も使える。遠藤さんの基本方針は「全部全体表示」。
- **出力先（固定）**：`~/Downloads/営業資料/claude提案資料/みらいジョブ掲載素材`（ツールの既定＝各自ホーム。全社ぶんここに集約）。
- **ファイル名**：`{正式社名}_画像{連番}.jpg`（例：`株式会社佐藤鉄工所_画像1.jpg`）。`--prefix` には「株式会社〇〇」の正式社名を渡す（スプレッドシートの会社名表記を使う）。
- **低解像度画像は自動で高画質化**（段階アップスケール＋シャープ化＋微補正）。元が極端に小さい場合(約2.6倍超)のみ「🔧高画質化適用＋差し替え推奨」と表示されるので、その場合だけ差し替えを促す。
- **任意**：ターミナルを使いたくない人向けに、ドラッグ＆ドロップ式のGUIアプリ（「みらいジョブ画像最適化.app」）を各自のデスクトップに作れる。作り方は同梱の `scripts/droplet.applescript` と README を参照（要：`~/mirai-job-tools/` にスクリプトをコピー）。

## B. インタビュー整形（先輩社員インタビュー＋1日の流れ）

### B-1. 回答データを読む
- 社員インタビューのスプレッドシート：
  `read_file_content` fileId=`1ik5dih0HYqT_XSDBRmNBDDnSraYKwwoLDJWet7GhRwo`（「みらいジョブページ作成インタビュー（回答）」。閲覧には共有Driveへのアクセス権が必要）
- このシートには2つの回答表がある：
  - **表1＝社員インタビュー**：お名前(イニシャル)/会社名/所属部署・役職/入社年(または勤続年数)/出身高校/卒業年/6:00〜18:00以降の各時間帯の業務/入社きっかけ/担当業務/やりがい/入社前後のギャップ/成長経験/今後挑戦したいこと/高校生へのメッセージ
  - **表2＝企業向けアンケート**：会社名/担当者/期待すること/教育体制/職種の魅力/雰囲気・価値観/PRポイント（※必要な時だけ補助的に使う）
- 対象企業名で表1の行を探す（会社名が「株式会社〇〇」表記の揺れに注意）。複数社員いれば各人ぶん作る。

### B-2. 出力フォーマット（厳守。下記の佐藤鉄工所が手本）
**Markdown(md式)** で2ブロック出力する。みらいジョブ編集ページに貼ると見出しがそのまま反映される。見出し階層は `#`＝ブロック名 / `##`＝社員名行 / `###`＝各見出し。
※ファイルは Macでダブルクリックで開けるよう **拡張子 `.txt`** で保存する（中身は純粋なMarkdown。拡張子は貼り付け内容に影響しない）。

**① 先輩社員インタビュー**
```
# 先輩社員インタビュー

## {所属部署・役職}　{お名前}さん（{卒業年の西暦}年入社）

### {見出し}

{本文（2〜3文の段落。長い内容は空行で段落を分ける）}

...
```
（入社年が「○年(勤続)」だけの時は「（勤続○年）」と書く。卒業年があればそちらを「○年入社」に優先利用）
- 見出しは Q&A を編集者目線で言い換える（例：「入社のきっかけ」→「モノづくりへの興味が入社のきっかけ」）。
- 標準の見出し順：**モノづくり…入社のきっかけ → 現在の仕事内容 → やりがいを感じる瞬間 → 成長を感じたこと → 今後の目標 → 高校生のみなさんへ**。
- きっかけの項では、現職（役職・担当）を1段落足して人物像が伝わるようにする（佐藤鉄工所の手本参照）。
- 回答が空の項目は見出しごと省く。

**② 先輩社員の1日の流れ**
```
# 先輩社員の1日の流れ

## {所属部署・役職}　{お名前}さん

### {時間帯}｜{ラベル}

{説明}

...
```
- 時間帯は **元の回答に出てくる区切りをできるだけ残す**（手本では 9〜12 と 13〜17 を別見出しにし、ラベルで内容を表現）。連続して全く同じ内容なら適宜まとめてよい。
- ラベルは内容から付ける（通勤・移動／出社・朝礼／成型機の組み立て・電気配線作業／昼休憩／午後の製造業務／終業・退社／帰宅 等）。
- 「なし」「移動中」など中身が薄い早朝枠は、自然な範囲で簡潔に（または省略）。
- 2ブロックの間は `---` で区切る。

### B-3. 文章ルール（最重要）
- 回答は短い箇条書き的に書かれている前提。**高校生が読んで働くイメージが湧くよう、丁寧で前向きな自然文に膨らませる**。
- ただし **事実を創作しない**。役職・年数・業務・数値・制度は回答どおり。書かれていない実績や数字を足さない。膨らませるのは言い回し・語り口のみ。
- 一人称は社員本人（「私」）。トーンは誠実で親しみやすく、過度に広告的にしない。
- 会社名は本文では「佐藤鉄工所」のように自然に。正式名称は見出し外で必要時のみ。

### B-4. 手本（佐藤鉄工所 I.Yさん）
入力（terse）：入社きっかけ「モノづくりに興味があった」／担当「成型機の組み立て」／やりがい「成型機が完成し、お客様より喜ばれるとき」／メッセージ「一緒に未来を切り拓いて行きましょう」、時間帯=9〜12と13〜17が同じ「実務業務（組み立てや電機配線）…」。
出力例は **同梱の `${CLAUDE_PLUGIN_ROOT}/skills/mirai-job-page/assets/株式会社佐藤鉄工所_インタビュー.txt`** を参照（この粒度・語り口・時間帯グルーピング・プレーンテキスト体裁に合わせる）。

## C. 修正依頼対応（先方からメールで届く）
先方の修正依頼は**メールで届く**。Claudeの役割は**修正後のブロックを貼り付け用に出力**すること（kosotsu-appは触らず、営業マンが貼る）。修正は「今ある内容」を「こう直す」なので、必ず次の3点を突き合わせる：

1. **修正依頼を把握**：Gmailコネクタで該当社のメールを検索・取得（or 担当者が内容を伝える）。「文章の修正・差し替え」「画像の差し替え・追加」「情報の追加」が主。
2. **現ページの内容・構成を取得**：公開ページ `mirai-job.jp/jobs/<id>` を読む（社名で検索 or URL指定）。**構成（承認済み）は変えず、中身だけ差し替える**のが原則。
   - 現ページ本文の確認は Claude in Chrome で `get_page_text` / DOMの img(alt=求人画像N) を見る。本文が画像スライドショー（`<!-- slideshow -->`＋`![求人画像N](…supabase…/job_images/…png)`）で組まれている会社もある。
3. **新素材を反映して修正ブロックを出力**：
   - 文章修正・差し替え → 該当箇所をmd式で出し「▼ここを差し替え」と位置明示。
   - 画像差し替え・追加 → 新画像を8:5最適化（A節）して掲載素材へ。「○枚目を差し替え」と指示。
   - 情報追加（新インタビュー等）→ 既存書式で新ブロック生成。
   - 出力末尾に「**何を・どこに・どう変えるか**」のチェックリストを必ず付ける。

### C-1. パンフレット（会社案内）が素材の会社（例：SVシステム）
一部の会社は社員インタビュー・1日・社長メッセージを**先方のパンフレットから**作る（従来は切り抜き画像を本文に貼っていた）。パンフレットは Illustrator `.ai` で届くことが多いが**中身はPDF互換**（ヘッダ `%PDF-1.6`）。手順：
- `.ai` を `.pdf` にコピーして扱う。`poppler`が無い環境では **Swift+PDFKit** でページを画像化（`page.draw(with:.mediaBox,to:ctx)`、scale 4xで高解像度）。
- 文章を**テキスト化**する場合は **macOS Vision OCR**（Swift `VNRecognizeTextRequest`, `recognitionLanguages=["ja","en"]`, `.accurate`）で正確に抽出 → md式テキストに整形。目視より圧倒的に正確。
- **画像で残す場合の切り抜きは担当者（ユーザー）が行う**（デザインを理解している人が切る方が確実なため、Claudeは原則クロップしない）。**Claudeの役割は、受け取った切り抜き画像を A節の `mirai_image_fit`（既定=contain＝背景ぼかしで8:5）にかけてサイズ調整し、低解像度なら自動高画質化すること**。
  - 切り抜き画像はDesktop等のスクショで渡されることが多い。スクショは小さい（〜400px）と3〜4倍拡大になり高画質化が走るが、文字なら実用十分に仕上がる。さらに鮮明にしたい時だけ「もっと高ズームで撮り直し」を促す。
- **ハイブリッド方針**（遠藤さんの既定）：インタビュー/1日/社長は**テキスト化（OCR）**、制度・図解・各インタビューパネルは**ユーザーが切り抜き→Claudeが8:5整形**。テキスト版と画像版を両方そろえれば貼り方を選べる。
- 命名例：`{社名}_社員IV_{氏名}.jpg`／`{社名}_社長メッセージ.jpg`／`{社名}_制度{n}_{制度名}.jpg`／`{社名}_事業内容{n}_{種別}.jpg`。氏名はパネルのキャプションで確定できることが多い。
- OCRで拾えない人名・固有名詞は「※要確認」と明記して本人確認に回す。

### C-2. 本文（企業からのメッセージ）の書式メモ
kosotsu-app編集ページの本文欄はMarkdown＋独自記法。画像スライドショーは
`<!-- slideshow -->` … `![求人画像N](画像URL)` を複数 … `<!-- /slideshow -->` で囲む。YouTube URLは専用欄に追加。テキスト見出しは #/##/### がそのまま反映される。

## 出力・報告
- インタビュー文は画面にも出し、画像と同じ `みらいジョブ掲載素材` フォルダに **`{正式社名}_インタビュー.txt`** として保存する。**中身はMarkdown(md式・上記#/##/###)**だが、拡張子は `.txt`（Macでダブルクリックで開けるようにするため。`.md`だと開けず「見れない」）。
- 最適化画像の枚数・保存先（みらいジョブ掲載素材）、低解像度の差し替え推奨、生成したインタビューの社員名を報告する。
- 「画像は編集ページ『求人画像』へ、本文は該当欄へ貼り付けてください。基本情報は求人票PDFを kosotsu-app にアップすれば自動入力されます」と案内する（③＝営業マンが手動で貼る工程）。
