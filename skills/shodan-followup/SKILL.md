# 商談後フォロー下書き作成スキル

株式会社高卒採用バンク（代表：遠藤隼聖）の商談後フォローを作る。対話セッション内で実行されるため、Gmail/Driveコネクタはそのまま使える。各商談につき **(a) フォローメールのGmail下書き** と **(b) 提案書PDF** の両方を作る（添付はコネクタ非対応なので、PDFは指定フォルダに保存し、送信者本人が下書きに手動ドラッグして送る）。

**このスキルは高卒採用バンクのメンバー全員が使える。** フォローメールは「実行した本人名義」で作る。名乗り・署名は本人のプロフィール（このスキルのディレクトリ内 `~/.claude/skills/shodan-followup/profile.json`）から差し込む。商談議事録の検知・宛先取得・Gmail下書き作成は、実行者本人が接続している Gmail/Drive コネクタ（＝本人のアカウント）に対して動く。

## 対象
- 引数が無ければ「本日」の商談を対象にする。
- 引数で会社名・日付が指定されたらそれを対象にする（例：「アームズ」「6/4」）。

## 手順

### 0. 送信者プロフィールを確定（初回のみ設定）
- 本人プロフィールは **`~/.claude/skills/shodan-followup/profile.json`** に置く（各メンバー自身のホーム配下。リポジトリには入らない＝個人別）。まずこのファイルを読む。形式：
```json
{ "name": "氏名（漢字）", "title": "役職", "email": "自分のメール", "phone": "電話番号" }
```
- **ファイルが無い／値が空の場合**：実行者に「氏名（漢字）・役職・メールアドレス・電話番号」を尋ね、その内容で `~/.claude/skills/shodan-followup/profile.json` を作成してから先に進む（次回以降は聞かない）。テンプレートは同ディレクトリの `profile.example.json` を参照。
- 以降の名乗り（書き出し）と署名は、この `profile.json` の値を差し込んで作る。会社名・HP・メディア掲載リンクは全員共通（固定）。

### 1. 当日の商談議事録を集める
- Gmail検索：`from:meetings-noreply@google.com newer_than:1d 会議の記録`（指定日があれば newer_than/after/before で調整）。
- 各メールの本文(HTML)に文字起こしDocのリンク `docs.google.com/document/d/<ID>` がある。<ID>を抽出する。
- **商談のみ対象**：タイトルが「【都道府県】会社名/担当者…」形式のもの。社内MTGやカジュアル面談（【都道府県】が付かないもの。例「久世さん」「セミナー…」「【カジュアル面談】…」）は除外。

### 2. 議事録を読む
- DriveのファイルIDで全文を読む（read_file_content）。

### 3. 空議事録はスキップ
- 相手が入室していない／バラシ／数十秒で終了など、実質的な会話が無いものは下書きを作らない（スキップ理由を記録）。

### 4. 宛先（予約担当者）を特定
- Gmailで「招待: 【…】会社名」や日程案内メールを検索し、本文の「メール:」や宛先から予約担当者のメールアドレスと姓を取得する。
- 実際の出席者が別人でも、件名・宛名・宛先は予約担当者にする。
- アドレスが特定できなければ下書きの宛先を空にし、その旨を報告に明記。

### 5. 重複防止
- その会社宛に本日すでに送信済み or 下書きがあればスキップ（search_threads / list_drafts で確認）。

### 6. フォローメール本文を作成（ルール厳守）
- 文字起こしは精度が低い前提で、固有名詞・専門用語を文脈と読みから正しい漢字に補正してから使う（例：絆創支援→伴走支援、ミラジブ→みらいジョブ、基準表→求人票、エ藤→遠藤）。
- **件名（固定）**：「{会社名（「株式会社」は外す）}様：先日のお打ち合わせのお礼と資料のご送付（高卒採用バンク）」
- **宛名**：1行目「株式会社{正式社名}」／2行目「{予約担当者の姓}様」
- **書き出し（固定。名前は profile.json の name を差し込む）**：
  お世話になっております。株式会社高卒採用バンクの{profile.name}です。
  先日はオンラインにて貴重なお時間をいただき、ありがとうございました。
- 商談に触れる語は必ず「先日」。「本日」「昨日」は使わない。
- **本文方針**：相手が漏らした“本音の課題”を相手の言葉で言い切る → 提案資料を「本メールに添付いたします」と伝える（本文で内容を説明しきらない）→ 商談で合意した次アクション/日付で締める（無ければ「◯日後に確認のご連絡をしてよいか」と期日を置く）。「いつでもご連絡ください」で終わらせない。
- 「明日中に資料をお送りします」等の“これから送る”予告は書かない（資料は添付前提）。
- 着地したプランがあればそれを前提に書く（社内稟議に通る“武器”として、決裁者にそのまま回せる体裁を意識）。
- **署名は本文末尾に必ず以下を入れる**（API作成の下書きにはGmail自動署名が付かないため）。`{profile.*}` は profile.json の値を差し込む。会社名・HP・メディア掲載リンクは全員共通で固定：

```
--
■□■──────────────────────■□■
株式会社高卒採用バンク
{profile.title} {profile.name}
TEL：{profile.phone}
Email：{profile.email}
HP: https://kousotsusaiyo-bank.jp/

＝＝弊社の取り組みが各種メディアに掲載されました==
日経新聞
https://www.nikkei.com/article/DGKKZO89479740Z10C25A6L61000/
中日新聞
https://biz.chunichi.co.jp/news/article/10/110975/
■□■──────────────────────■□■
```

### 6.5 提案書PDFを生成（各商談ごと）
- 白基調・6ページ固定構成のジェネレーターを使う：
  `python3 ~/.claude/skills/shodan-followup/gen_deck.py <content.json> "<出力先.pdf>"`
- 出力先フォルダ：`~/Downloads/営業資料/claude提案資料/`。無ければ作成する：`mkdir -p ~/Downloads/営業資料/claude提案資料`（`~` は実行者のホーム＝各メンバー自身のDownloads）。
- ファイル名：`{正式社名}御中_高卒採用のご提案書.pdf`（例：`株式会社ザ・チューブ御中_高卒採用のご提案書.pdf`）
- 事前に：`pip3 install --user --quiet reportlab Pillow`（未インストール時のみ）
- **料金・プラン・市場数値・事例・サービス内容は、同梱の `service_reference.md`（このスキルのディレクトリ内）を“正”とする。** 着地プランの価格は必ずここから引く（みらいジョブ：ベーシック56万円／スタンダード98万円／プレミアム134万円・いずれも**年度・税抜**）。ここに無い数値は創作しない。
- content.json のスキーマ（議事録から各値を埋める。誤字補正済み・**着地プランのみ**・数値は service_reference.md／議事録準拠で**創作しない**）：
```json
{ "kicker":"高卒採用 ご提案資料",
  "cover_title":"1〜2行のタグライン（\\nで改行可）",
  "cover_lead":"表紙のリード一文",
  "cover_client":"{正式社名} 様",
  "issues_heading":"見出し", "issues_lead":"リード文",
  "issues":[["◎","課題名","説明3-4文"],["¥","",""],["◆","",""]],
  "bg_heading":"見出し",
  "causes":[["原因名","説明"],["",""],["",""]],
  "bg_conclusion":"つまり〜の一言",
  "sol_heading":"見出し",
  "solutions":[{"sub":"知られる","title":"みらいジョブ掲載","items":["","",""]},{"sub":"伝わる","title":"","items":["","",""]}],
  "data":[["数値","ラベル"],["",""],["",""]],
  "plan_heading":"見出し",
  "plan":{"name":"スタンダード","price":"98万円","unit":"／年度（税抜）","desc":"説明","badge":"短い訴求","included":[["項目","説明"],["",""],["",""]]},
  "plan_note":"おすすめ理由＋※税抜の注記",
  "close_heading":"次のステップ","close_lead":"リード文",
  "close_case":"近い事例があれば1文。無ければ空文字 \"\"",
  "steps":[["STEP 1","見出し","説明"],["STEP 2","6/XX(曜) お電話","説明"],["STEP 3","",""]] }
```
- 値の確証が無い数値は創作しない（定性表現にするか省く）。着地プランが1つに決まっていればそれのみを plan に入れる。生成後、出力PDFのパスを控えて報告に含める。

### 7. Gmail下書きを作成
- create_draft で、宛先=予約担当者アドレス、上記の件名・本文で下書きを作成する（添付はコネクタ非対応のため本文のみ。資料は送信者本人が手動添付）。下書きは実行者本人のGmailに作られる。

### 8. 結果を報告
- 当日の商談件数、作成した下書き（会社名・宛先メール）、生成した提案書PDFのファイル名、スキップしたもの（会社名と理由）を一覧で出す。
- 「提案書PDFは `~/Downloads/営業資料/claude提案資料/` に保存しました。送信前に下書きへ手動でドラッグ添付してください」と添える。

詳細な背景・料金・サービス内容・事例は、同梱の `service_reference.md`（最新サービス資料 2026年版から作成）を参照する。
